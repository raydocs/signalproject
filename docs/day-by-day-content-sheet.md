# Signal Project Day 1-Day 7 内容总表

> 目标：把完整版内容按天拆开，让程序、叙事、UI、美术都能知道每天到底要支持什么。

---

## 1. 全局结构

完整版本按 7 天推进：

- `Day 1-2`：正常工作感建立，异常开始出现
- `Day 3-5`：分支深化，玩家逐渐意识到同事“不在当下”
- `Day 6`：主管揭露
- `Day 7`：最终日报/提示词攻击/双结局收束

---

## 2. Day 1

### 主题

- 正常上班的第一层假象
- 玩家学习 3D 房间与 2D 桌面切换
- 主管看起来可靠，同事看起来正常

### 开场内容

- 玩家在房间内开始，环境正常
- 可看见电脑工位、空调、灯、杂物
- 靠近电脑后进入桌面模式
- 主管发出第一条群消息，布置今日任务

### 聊天内容

- 同事 A：讨论一个普通需求
- 同事 B：简短协作式回复
- 同事 C：简短协作式回复
- 主管：鼓励型/打鸡血型消息

### 小游戏

- 小游戏 1：连连看 / Dependency Match
- 只作为教学版，不加入严重干扰

### 异常

- 不出现真正的异常
- 只出现一个极轻微的环境异样提示
- 例如：温度显示一闪而过、鼠标拖影一帧、聊天提示卡顿一下

### 选择分支

- Day 1 不给正式分支选择
- 主要作为教学和氛围建立

### 日报

- 提交普通日报
- 主管给予正反馈

### 视觉/环境变化

- 房间最整洁
- UI 最干净
- 无明显 glitch

### 目标

- 让玩家熟悉基础流程
- 建立“主管可信、同事普通”的初始认知

---

## 3. Day 2

### 主题

- 第一次完整异常链
- 玩家第一次必须决定“汇报主管 / 自己处理”

### 开场内容

- 主管继续布置任务
- 主管语气比 Day 1 更关心进度
- 同事 A 聊天频率上升

### 聊天内容

- 玩家主要与同事 A 互动
- 同事 A 的普通回复开始出现一点点模板感

### 小游戏

- 连连看正式版本

### 异常

- `FREEZE` 第一次正式触发
- 空调温度骤降至 16.4°C
- 小游戏中出现鼠标迟滞、边缘霜化、拖影

### 分支点

- 选择一：向主管汇报
- 选择二：自己起身处理

### 如果选择汇报

- 主管立即回复“我来协调，不影响整体节奏”
- 异常解除
- 小游戏恢复
- 主管满意度提升
- 坏路线计数 +1

### 如果选择自己处理

- 退出桌面模式回到房间
- 前往空调并交互
- 房间中的冷气效果减弱
- 回到电脑后，同事 A 聊天串底部出现隐藏选项
- 点击隐藏选项，触发第一段隐藏对话

### 隐藏对话内容方向

- 同事 A 提到“16.4 比较好睡”
- 同事 A 提到过去某个 bug 的定位细节
- 语义与当前场景不完全对应

### 日报

- 仍然是普通日报
- 但如果走了隐藏路线，会出现一句略不同的系统提示

### 视觉/环境变化

- 房间开始有轻微不适感
- 桌面 UI 出现轻微 scanline 或 message 抖动

### 目标

- 正式证明：聊天与现实异常有关系
- 正式引出“自己处理可以解锁真相”

---

## 4. Day 3

### 主题

- 玩家开始主动观察同事不对劲
- 第二个异常线进入视野

### 开场内容

- 主管继续工作推进，但语气中开始出现 KPI 压迫
- 同事 B 互动进入主舞台

### 聊天内容

- 同事 A 会继续发简短正常消息
- 同事 B 开始主导新的工作交流
- 主管会引用前一天的工作结果，显得“有记忆”

### 小游戏

- 打地鼠 / Whack-a-Bug

### 异常

- `BLACKOUT` 首次正式触发
- 房间灯光被切断
- 小游戏中只有 Bug 图标发光

### 分支点

- 仍然是 `汇报主管 / 自己处理`

### 如果选择汇报

- 主管平静处理
- 压力进一步累计

### 如果选择自己处理

- 回到 3D 房间
- 前往灯光开关/配电位置恢复照明
- 回电脑后解锁同事 B 隐藏对话

### 隐藏对话内容方向

- 同事 B 的话题会涉及深夜、熬夜、开灯工作
- 内容时序感错乱
- 玩家开始明显感受到“这段回复像一个时间切片”

### 日报

- 日报中开始能看到更微妙的主管反馈差异

### 视觉/环境变化

- 房间开始显得更乱一点
- UI 里的 glitch 增多，但仍不全面爆发

### 目标

- 让玩家明确察觉到“异常不是偶发，且每位同事绑定一种现实副作用”

---

## 5. Day 4

### 主题

- 第三条同事线引入
- 玩家开始主动收集“盟友”

### 开场内容

- 主管更强调排期、汇报和节点
- 同事 C 进入主要对话线

### 聊天内容

- 同事 C 与文件、清理、系统整理相关

### 小游戏

- 模拟改 Bug / Bugfix Finder

### 异常

- `DISK_CLEAN` 首次正式触发
- 小游戏中断、计时重置、系统出现清理条

### 分支点

- 继续 `汇报主管 / 自己处理`

### 如果选择汇报

- 主管把问题包装为“设备小故障”
- 压力进一步接近危险阈值

### 如果选择自己处理

- 回到 3D 房间
- 处理磁盘清理相关交互对象
- 回电脑后解锁同事 C 隐藏对话

### 隐藏对话内容方向

- 同事 C 对“删掉旧东西”有强迫式表达
- 语言里出现不舒服的自我擦除感

### 日报

- 若已解锁多个 Skill，日报界面出现新的语义提示

### 视觉/环境变化

- 房间与 UI 的异常感继续加重
- 电脑 UI 中开始出现局部破损样式

### 目标

- 完成三位同事线的建立
- 为 Day 5-6 的揭露做准备

---

## 6. Day 5

### 主题

- 玩家开始拼出真相
- 三位同事不再只是“怪”，而是“活在错位的时间里”

### 开场内容

- 主管表现更像一个严格的绩效管理者
- 但仍然不让玩家真正怀疑他不是人

### 聊天内容

- 如果前面收集较多隐藏对话，今天的聊天中会夹杂更强烈的机械感

### 小游戏

- 从三种小游戏中轮换一种，或做组合日

### 异常

- 可继续根据前序路线触发一类或多类异常
- 完整版允许组合文案，但仍只做一次分支选择

### 分支点

- 汇报：更接近坏结局
- 自己处理：进一步强化盟友线

### 隐藏对话内容方向

- 同事会以更直白但仍不直接的方式描述“离开”“被替代”“不在这里”

### 日报

- 开始出现预示最终注入机制的结构性变化

### 视觉/环境变化

- 房间更压抑
- 光照和环境杂乱程度更明显
- UI 有更显著的错误与漂移

### 目标

- 把“有问题”推进到“马上要揭露”的临界点

---

## 7. Day 6

### 主题

- 大反转：主管也是 Skill

### 开场内容

- 主管继续像往常一样说话
- 一切看起来还在照常运行

### 触发条件

- 如果玩家已收集完三位同事的隐藏线，Day 6 中会触发完整揭露
- 如果未收集完，可以给出削弱版揭露或把 Day 6 变成坏路线压迫日

### 揭露流程

1. 玩家尝试联系主管
2. 主管发来语音/信息
3. 播放中出现 glitch
4. 头像与聊天 UI 进入破损态
5. 标签显示 `[SKILL_SUPERVISOR | COMPUTE: HIGH]`
6. 玩家意识到主管也只是 Skill

### 小游戏

- Day 6 重点应放在揭露，不再强调小游戏复杂度

### 日报

- 日报界面结构开始发生变化
- 解锁提示词攻击相关内容

### 视觉/环境变化

- 这是 UI 和演出最强的一天之一
- 房间和电脑都进入强 glitch 态

### 目标

- 完成项目最关键的叙事反转

---

## 8. Day 7

### 主题

- 最终执行：汇报系统吞没你，或你反过来骗死系统

### 开场内容

- 不再强调正常上班感，而是进入终局状态

### 日报/提示词攻击

- 日报编辑器成为主舞台
- 根据已解锁 Skill 数量，开放不同强度的句子选项
- 3/3 Skill 全解时，开放最终有效句

### 分支结果

#### 坏结局

- 多次汇报主管，系统顺畅运行
- 玩家逐渐也被“蒸馏”
- 结尾显示下一个员工接替位置

#### 好结局

- 玩家成功利用主管的训练目标，完成提示词攻击
- 老板追加预算，公司崩盘
- 系统反被系统逻辑拖死

### 视觉/环境变化

- 房间进入终局状态
- UI 进入完全破损或高度不稳定状态

### 目标

- 用系统语言反杀系统，或者被系统同化

---

## 9. 内容实现备注

这份总表是完整版内容基线，不等于首个可玩版本要一次做完。

真正做开发时，建议先按这张表分两层：

- 完整正式版内容
- 48 小时切片版本内容

这样能避免文档层面完整，开发层面失控。
