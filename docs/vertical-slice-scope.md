# Signal Project 48 小时竖切片范围文档

> **竖切片权威说明（Authoritative for Slice）**
> - 这份文档与 `docs/game-state-machine.md`、`docs/interaction-spec.md` 一起构成 `48 小时竖切片` 的最高优先级运行契约。
> - 若它们与其他“完整版 / 参考性”文档冲突，以这三份文档为准。
> - 本轮冻结的是 **竖切片可执行边界**，不是完整版设计边界。

---

## 1. 切片目标

48 小时版本不是完整 7 天正式版，而是一个 `可从开场玩到结尾、完整体现核心机制和气质的竖切片`。

这个切片必须让玩家体验到：

- 远程办公 / 家中工位 / 独处空间的 3D 压迫感
- 进入电脑后的 2D 桌面与聊天交互
- 聊天即调用 Skill、异常会在现实空间体现
- 玩家在“汇报主管”与“自己处理”之间做选择
- 自己处理会解锁隐藏对话
- 至少一个完整异常链跑通
- 至少一个可以成立的结局收束

---

## 2. 权威切片边界（本次冻结结果）

### 2.1 这次真正要交付的运行范围

竖切片的权威运行边界固定为：

- `Day 1`：开场 / 电脑进入 / 基础聊天 onboarding
- `Day 2`：唯一一条 **完整可执行异常链**
- `Day 3`：简化结尾包装与结果落点

换句话说，本次切片不是“做 Day 1-Day 7 的缩略版”，而是：

- 用 `Day 1` 建立规则
- 用 `Day 2` 完整证明核心循环
- 用 `Day 3` 给出可收束的反馈与结尾

### 2.2 Day 级别执行规则

#### Day 1

必须完成：

- 玩家进入房间与电脑交互
- 玩家进入 `DesktopIdle / ChatActive`
- 玩家理解“聊天是主要推进方式”
- 建立主管与同事 A 的基础存在感

明确不要求：

- 执行完整异常处理链
- 解锁隐藏对话
- 引入第二种异常

#### Day 2

必须完成并且必须跑通：

- 聊天推进
- 小游戏启动
- `FREEZE` 触发
- `Report Supervisor / Handle Myself` 二选一
- 若选择 `Handle Myself`，则必须回 3D 房间处理空调
- 异常解决后回电脑
- 隐藏选项出现
- 隐藏对话触发
- 进入 `ReportPhase`

这是 **竖切片唯一强制完整实现的异常日**。

#### Day 3

作用固定为：

- 承接 Day 2 路线结果
- 展示主管可疑感 / 真相碎片的简化反馈
- 进入简化好/坏结尾收束

明确不要求：

- 新异常类型执行
- 新的完整 3D 处理链
- 完整 Day 4-Day 7 内容

## 3. 命名冻结（Canonical Naming Freeze）

以下命名自本文件起冻结，后三份切片权威文档必须完全一致：

| 类别 | 权威命名 | 说明 |
|---|---|---|
| 路线枚举 | `E_RouteBranch` | 不再使用 `E_RouteType` |
| 路线值 | `Neutral` / `ReportSupervisor` / `HandleMyself` | 与按钮语义一一对应 |
| 异常枚举值 | `DISKCLEAN` | 不再使用 `DISK_CLEAN` |
| 结尾字幕表 | `DT_EndingSubtitles` | 不再使用 `DT_EndingSequences` |
| 异常按钮文案 | `Report Supervisor` / `Handle Myself` | 切片中固定使用这组英文按钮标签 |

### 3.1 异常名的切片执行边界

`E_AnomalyType` 可以保留未来扩展值：

- `None`
- `FREEZE`
- `BLACKOUT`
- `DISKCLEAN`

但对 48 小时竖切片而言，**真正执行的只有**：

- `None`
- `FREEZE`

以下两项仅允许作为“未来内容占位 / 名字存在 / 图标存在 / 文案占位”出现：

- `BLACKOUT`
- `DISKCLEAN`

它们在本次切片中 **不触发、不进入 3D 处理、不参与分支、不影响结尾判定**。

## 4. 竖切片唯一权威主循环

必须完整跑通下面这条主链：

1. 玩家在 3D 房间内醒来/站在工位前
2. 靠近电脑，进入桌面模式
3. 和同事 A 进行聊天
4. 聊天后开始一个小游戏
5. 小游戏过程中触发 `FREEZE`
6. 弹出选择：`Report Supervisor / Handle Myself`
7. 若选择 `Report Supervisor`
   - 记录路线为 `ReportSupervisor`
   - 当前 `FREEZE` 立即视为“系统已接管并解决”
   - 玩家 **不离开电脑**
   - 当天流程继续，小游戏恢复/收束后进入 `ReportPhase`
   - **不会** 解锁隐藏对话
8. 若选择 `Handle Myself`
   - 记录路线为 `HandleMyself`
   - 退出桌面，进入 `HandleAnomaly3D`
   - 玩家必须前往空调交互并解除 `FREEZE`
   - 异常未解除前，电脑不可重新进入
   - 异常解除后，允许返回电脑
   - 聊天中出现隐藏选项
   - 点击隐藏选项，触发隐藏对话
   - 之后进入 `ReportPhase`
9. 根据 Day 2 路线结果进入 Day 3 简化结尾包装
10. 播放简化好/坏结尾反馈

### 4.1 分支结果冻结

#### Report Supervisor 分支

权威结果固定为：

- 异常被立即结案
- 日程继续推进，不跳过当日流程
- 玩家保持在桌面链路内
- 小游戏恢复或立即收束，然后进入 `ReportPhase`
- 不解锁隐藏选项
- 结尾倾向于“服从系统 / 坏结局反馈”

#### Handle Myself 分支

权威结果固定为：

- 玩家必须进入 3D 处理异常
- 目标交互物固定为 `空调`
- 只有在 `FREEZE` 被玩家亲自解决后，隐藏选项才可见
- 隐藏对话只能发生在“处理完成之后、进入 Report 之前”
- 结尾倾向于“自我处理 / 真相靠近 / 好结局反馈”

## 6. 竖切片内“真正执行”的对象边界

> 这里冻结的是 **执行行为边界**。更细的拥有者/引用关系在后续 wiring / responsibility 文档中细化，但不能违背这里的运行边界。

| 对象 | 切片中的权威行为 |
|---|---|
| `BP_ComputerTerminal` | 只负责从 `RoomExplore` 进入桌面；若玩家处于未解决的 `HandleAnomaly3D`，则禁止重新进入电脑 |
| `BP_AirConditionerUnit` | 只负责处理 `FREEZE`；是切片唯一必做的 3D 异常解决物 |
| `WBP_DesktopRoot` | 承载桌面模式下的 app 切换与可见内容 |
| `WBP_ChatApp` | 承载同事 A 正常聊天、隐藏选项出现与隐藏对话触发 |
| `WBP_MG_DependencyMatch` | 切片唯一必做小游戏；`FREEZE` 在该阶段触发 |
| `WBP_AnomalyChoicePopup` | 只提供 `Report Supervisor / Handle Myself` 两个分支按钮 |
| `WBP_ReportEditor` | 作为切片的简化日报/结算提交界面 |
| `WBP_EndingTitleCard` | 显示切片简化结尾反馈 |
| `BP_AnomalyManager` | 只需要真正管理 `FREEZE` 的激活与解除；其他异常仅保留占位值 |
| `BP_RouteStateManager` | 只需要记录切片所需的一次关键路线结果 |
| `BP_ChatConversationManager` | 推进聊天、积累 side effect、在条件满足后暴露隐藏选项 |
| `BP_MinigameManager` | 启动并关闭切片小游戏，承接异常中断 |
| `BP_HiddenDialogueUnlocker` | 在“自己处理成功”后解锁隐藏选项 |

### 6.1 本轮不作为切片执行边界的对象

以下对象即使出现在其他完整版文档里，也 **不是本轮三份权威文档要冻结的执行核心**：

- `BLACKOUT` / `DISKCLEAN` 对应的 3D 处理物
- 第二、第三小游戏
- Day 4-Day 7 专用剧情对象
- 完整 RevealSequence 专用流程对象
- 任何要求多异常并行的系统

## 7. 输入与锁定规则（切片范围层）

以下规则在三份权威文档中统一生效：

- 玩家只有在“正常桌面状态”下才能主动离开桌面
- `AnomalyChoice` 期间不允许离开桌面
- 进入 `HandleAnomaly3D` 且异常未解除时，电脑重新进入被锁定
- `Handle Myself` 路线下，必须先处理空调，后返回电脑
- 隐藏选项只在 `Handle Myself` 且 `FREEZE` 已解除后出现
- `Report Supervisor` 路线不会触发隐藏对话

## 9. 成功标准

本次竖切片唯一正确判断标准仍然是：

- 玩家能不能完整体验到 `聊天 -> 异常 -> 选择 -> 起身处理 -> 解锁隐藏真相`

并且现在要补上一条更具体的实现标准：

- 团队在 Unreal 中创建最终资产和蓝图时，不需要再猜测 `哪一天真实执行、哪个异常真正执行、按钮和枚举到底叫什么、两条路线各自到底落到哪里`

只要这一点成立，这份范围文档才算真正可用于 48 小时开工。
