# Signal Project 48 小时竖切片范围文档

> 目标：定义 `48 小时 Game Jam 可交付版本` 的真实范围。
> 原则：保留项目灵魂，不追求完整版内容覆盖。

---

## 1. 切片目标

48 小时版本不是完整 7 天正式版，而是一个 `可从开场玩到结尾、完整体现核心机制和气质的竖切片`。

这个切片必须让玩家体验到：

- 远程办公 / 家中工位 / 独处空间的 3D 压迫感
- 进入电脑后的 2D 桌面与聊天交互
- 聊天即调用 Skill、异常会在现实空间体现
- 玩家在“汇报主管”与“自己处理”之间做选择
- 自己处理会解锁隐藏对话
- 至少一个完整异常链跑通
- 至少一个可以成立的结局收束

---

## 2. 48 小时版本定位

### 2.1 版本定位

这是一个 `Playable Vertical Slice`，不是 Demo Reel，也不是概念样机。

它必须满足：

- 有开始
- 有主循环
- 有关键分支
- 有至少一次真相揭露
- 有收尾

### 2.2 成功标准

玩家第一次游玩时，能够清楚理解：

- 这个世界的玩法结构
- 聊天和异常之间的关系
- 为什么要自己起身去处理异常
- 为什么隐藏对话重要
- 主管并不可信

---

## 3. 本次必须完成的内容

## 3.1 核心玩法闭环

必须完整跑通下面这条主链：

1. 玩家在 3D 房间内醒来/站在工位前
2. 靠近电脑，进入桌面模式
3. 和同事 A 进行聊天
4. 聊天后开始一个小游戏
5. 小游戏过程中触发 `FREEZE` 异常
6. 弹出选择：`汇报主管 / 自己处理`
7. 如果选择 `自己处理`
8. 玩家退出桌面模式，回到 3D 房间
9. 前往空调并交互，解除异常
10. 返回电脑
11. 聊天框出现隐藏选项
12. 点击隐藏选项，触发隐藏对话
13. 进入一个简化日报/结算界面
14. 根据选择进入一个结尾反馈

这条链必须是 `可从头到尾独立完成` 的。

---

## 3.2 3D 部分必须完成的范围

### 必做

- 1 个可探索房间
- 1 个玩家角色
- 1 台可交互电脑
- 1 个可交互空调
- 基础环境灯光
- 异常发生时的房间降温/压迫反馈
- 桌面模式与房间模式来回切换

### 可接受的简化

- 房间只做一个主空间，不做多个可进入房间
- 不做复杂开门系统
- 不做复杂拾取系统
- 不做角色身体动画系统，只做镜头和输入切换

---

## 3.3 2D 部分必须完成的范围

### 必做

- 桌面主界面
- 聊天界面
- 1 个小游戏界面
- 异常选择弹窗
- 隐藏选项显示
- 隐藏对话界面
- 简化日报/结算界面

### 可接受的简化

- 聊天输入可以是假输入，走预设选项或固定发送按钮
- 日报编辑器可以做成简化版“句子选择面板”
- 不需要可拖拽窗口系统
- 不需要完整 OS 文件系统模拟

---

## 3.4 叙事范围

48 小时版本建议只覆盖 `Day 1-Day 3` 的内容量，但要拼成一个完整体验。

### 推荐结构

- `Day 1`：正常工作 + 轻微不适感建立
- `Day 2`：第一次完整异常链 + 自己处理后解锁隐藏对话
- `Day 3`：主管可疑感提升 + 一个简化收束结尾

### 结尾策略

竖切片不强求完整 Day 7 终局，但必须有一个明确的临时收束：

- 方案 A：用“主管第一次露出机械痕迹”作为切片结尾
- 方案 B：用“同事 A 说出关键真相碎片”作为切片结尾
- 方案 C：做简化好/坏双结尾，根据第一次选择给出不同收束画面

推荐使用 `方案 C`，因为最有可玩性。

## 4. 本次明确只做这些系统

## 4.1 异常系统

### 必做

- `FREEZE` 完整实现

### 不进 48 小时主线，但可做假入口

- `BLACKOUT` 只留名字、图标、占位提示
- `DISK_CLEAN` 只留名字、图标、占位提示

理由：

- 一个异常做完整，比三个异常都半成品更有价值

## 4.2 小游戏系统

### 必做

- `连连看 / Dependency Match` 或者其极简变体

### 不做

- `打地鼠`
- `模拟改 Bug`

理由：

- 先用一个小游戏打通 `聊天 -> 异常 -> 选择 -> 处理 -> 解锁` 的闭环

## 4.3 同事系统

### 必做

- 同事 A 完整链路
- 主管基础存在感

### 不做完整内容

- 同事 B 只做占位
- 同事 C 只做占位

## 4.4 结局系统

### 必做

- 一个简化坏结局反馈
- 一个简化好结局反馈

### 不做

- 完整公司崩盘 montage
- 完整 Day 7 提示词攻击系统

## 9. 这份切片范围的最终结论

48 小时版本的正确目标不是：

- “把完整文档全做出来”

而是：

- “把 Signal 最核心的那一刀切出来，让人一玩就知道这个项目成立”

所以本次切片的唯一正确判断标准是：

- 玩家能不能完整体验到 `聊天 -> 异常 -> 选择 -> 起身处理 -> 解锁隐藏真相`

只要这个成立，竖切片就是成功的。
