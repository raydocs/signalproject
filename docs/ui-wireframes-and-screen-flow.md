# Signal Project UI 线框和页面流

> 目标：定义核心页面结构、信息层级和跳转逻辑，作为 UMG 与 UI 美术的统一依据。

---

## 1. UI 总体原则

这个项目 UI 不是“传统菜单”，而是游戏世界的一部分。

设计原则：

- 桌面 UI 要像真实远程办公系统
- 信息密度高，但布局要清晰
- 正常状态克制、冷静、企业化
- 异常状态通过轻微破坏逐步升级
- 页面切换不能太花，要有“在电脑里操作”的感觉

## 3. 线框定义

## 3.1 启动画面

### 页面目标

- 建立“工作系统启动”的感觉
- 完成进入游戏前的 mood 铺垫

### 结构

```text
┌──────────────────────────────────────┐
│ SIGNAL REMOTE WORK OS                │
│ booting...                           │
│                                      │
│ [progress bar / status text]         │
│                                      │
│ [Press any key / Continue]           │
└──────────────────────────────────────┘
```

### 要点

- 不做传统“游戏主菜单”感
- 更像一个公司工作系统在启动

---

## 3.2 房间交互提示 UI

### 页面目标

- 在 3D 场景中提供极简交互提示

### 结构

```text
          [crosshair]

   [E] Use Computer

[optional objective text at top-left]
```

### 要点

- 只显示必要信息
- 不要有太强 HUD 游戏感

---

## 3.3 桌面主界面

### 页面目标

- 作为桌面模式的 Hub
- 统一承载聊天、小游戏、日报入口

### 结构

```text
┌────────────────────────────────────────────────────┐
│ Top HUD: Day / temp / brightness / disk / alerts   │
├────────────────────────────────────────────────────┤
│                                                    │
│                Desktop Wallpaper                   │
│                                                    │
│   [Chat App]   [Tasks]   [Report]   [System]       │
│                                                    │
│                                                    │
├────────────────────────────────────────────────────┤
│ Status strip / notification area                   │
└────────────────────────────────────────────────────┘
```

### 要点

- 图标数量不要太多
- 默认让 `Chat App` 最显眼

---

## 3.4 聊天界面

### 页面目标

- 作为日常叙事核心页面

### 结构

```text
┌────────────────────────────────────────────────────────────┐
│ Chat App Header                                           │
├───────────────┬────────────────────────────────────────────┤
│ Contact List  │ Current Thread                             │
│               │                                            │
│ A             │  message bubble                            │
│ B             │  message bubble                            │
│ C             │  message bubble                            │
│ Supervisor    │  hidden option card                        │
│               │                                            │
├───────────────┴────────────────────────────────────────────┤
│ Input bar / send / preset phrase area                     │
└────────────────────────────────────────────────────────────┘
```

### 信息层级

- 当前联系人名称
- 对话线程
- 隐藏选项卡
- 输入和发送

### 异常态变化

- 头像轻微错位
- 打字提示抖动
- 隐藏选项卡有独特冷蓝视觉

## 3.8 异常选择弹窗

### 页面目标

- 在紧张时刻给出明确决策

### 结构

```text
┌──────────────────────────────────────┐
│  WARNING: TEMPERATURE ANOMALY        │
│                                      │
│  Air conditioner dropped to 16.4 C   │
│                                      │
│  [Report Supervisor] [Handle Myself] │
└──────────────────────────────────────┘
```

### 要点

- 中心弹出
- 底层小游戏暂停但可见

---

## 3.9 隐藏对话界面

### 页面目标

- 让玩家感觉是在“正常聊天界面里看到错误的东西”

### 结构

还是聊天界面，但有这些变化：

- 当前线程背景轻微噪点
- 消息文字短暂乱码后恢复
- 隐藏气泡样式与普通消息不同

```text
[A]

 normal message
 hidden fragment message
 hidden fragment message
 hidden fragment message
```

---

## 3.11 主管揭露界面

### 页面目标

- 把 UI 本身变成反转演出的一部分

### 结构

```text
Chat thread with supervisor

voice card playing...
glitch...
portrait corrupts...
[SKILL_SUPERVISOR | COMPUTE: HIGH]
```

### 要点

- 不需要复杂新页面，本质上可以是聊天界面的一次特殊状态

---

## 3.12 结局界面

### 页面目标

- 用最低成本给出清晰而强烈的收束

### 结构

```text
┌──────────────────────────────────────┐
│ ENDING TITLE                         │
│                                      │
│ subtitle / final narrative lines     │
│                                      │
│ [Restart] [Quit]                     │
└──────────────────────────────────────┘
```

## 4. 页面流

## 4.1 主流程页面流

```text
Boot
 -> RoomExplore
 -> DesktopRoot
 -> Chat
 -> Minigame
 -> AnomalyChoice
    -> Report Supervisor -> Minigame Resume -> Report
    -> Handle Myself -> RoomExplore(Anomaly) -> Chat Hidden Dialogue -> Report
 -> Day Transition
 -> next day
```

## 4.2 终局流程页面流

```text
DesktopRoot
 -> Chat / Reveal
 -> ReportEditor
 -> Detection / Success
 -> EndingScreen
```
